# Security Policy

Ecritum has no public stable release yet. Do not report exploitable issues in
public issues.

Until the public repository intake is live, report suspected vulnerabilities to
the project maintainer through the private channel used for release
coordination.

The monitored intake path for the first public release is GitHub private
vulnerability reporting for the Ecritum repository. The project owner owns that
repository setting and monitoring path for the first Community release.

Supported versions and vulnerability response rules are defined in
`docs/security/vulnerability-response.md` and enforced by
`scripts/check-vulnerability-response.py`.
