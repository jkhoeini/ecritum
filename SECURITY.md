# Security Policy

Ecritum has no public stable release yet. Do not report exploitable issues in
public issues.

Until public release infrastructure exists, report suspected vulnerabilities to
the project maintainer through the private channel used for release
coordination. Before the first public release, this file must name a monitored
security intake path such as private GitHub Security Advisories or a dedicated
security mailbox.

Supported versions and vulnerability response rules are defined in
`docs/security/vulnerability-response.md` and enforced by
`scripts/check-vulnerability-response.py`.
